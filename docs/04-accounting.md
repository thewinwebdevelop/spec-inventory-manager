# 04 — Cloud Accounting (ไทย)

เชื่อม "สต๊อก" กับ "บัญชี" ให้กำไร/ขาดทุนแม่นจริง และออกเอกสารตามมาตรฐานไทยได้

> ⚠️ ความถูกต้องทางบัญชี/ภาษีพลาดไม่ได้ — ทุกการคำนวณต้องมี test และควรให้นักบัญชีจริงรีวิวก่อน production

## 1. แนวคิดหลัก: ต้นทุนที่ระดับ component
สินค้าเซต (เสื้อ+กระโปรง) ซื้อเข้าแยกชิ้น แยก supplier แยกใบเสร็จ
→ ต้นทุนอยู่ที่ `InventoryItem` (เสื้อ, กระโปรง) ไม่ใช่ที่ "เซต"
→ ขายเซต = COGS รวมจาก component → กำไรต่อออเดอร์แม่น

## 2. วงจรเอกสาร → สต๊อก → บัญชี

```
ซื้อเข้า (PurchaseDocument)
   └▶ PURCHASE_IN movement ต่อ InventoryItem + อัปเดต avgUnitCost (weighted-avg)
ขาย (Order)
   └▶ SALE_OUT movement + บันทึก COGS = Σ(component.avgUnitCost × qty)
   └▶ รายรับ = ยอดขาย, หักค่าธรรมเนียม platform/ค่าส่ง
P&L = รายรับ − COGS − ค่าใช้จ่าย − ค่าธรรมเนียม
```

## 3. การคิดต้นทุน (costing)
- วิธี: **weighted-average ต่อ InventoryItem** (เริ่มต้น; เปิดทาง FIFO อนาคต)
- ซื้อเข้า: `newAvg = (onHand·oldAvg + qtyIn·costIn) / (onHand + qtyIn)`
- ต้นทุนนำเข้า (landed cost) รวมค่าส่ง/ค่าธรรมเนียมเข้าต้นทุนได้ (ตั้งค่าได้)

## 4. เอกสารที่ออกได้ (`AccountingDocument`)
| ประเภท | ใช้เมื่อ |
|--------|---------|
| `invoice` / ใบกำกับภาษี | ขาย/เรียกเก็บ (พร้อม VAT ถ้าจด VAT) |
| `receipt` / ใบเสร็จรับเงิน | รับเงินแล้ว |
| `payment_voucher` / ใบสำคัญจ่าย | บันทึกรายจ่าย |
| `substitute_receipt` / ใบรับรองแทนใบเสร็จรับเงิน | รายจ่ายที่ไม่มีใบเสร็จจากผู้ขาย |
| `purchase_doc` | นำเข้าสต๊อก (ผูก PURCHASE_IN) |

ข้อกำหนดเอกสาร: เลขรันต่อเนื่อง (running no.), ข้อมูลผู้ออก/ผู้รับ, รายการ, ฐานภาษี, VAT, ยอดสุทธิ, PDF ฟอนต์ไทย

## 5. ภาษีไทย (ขอบเขตที่ต้องเผื่อ)
- **VAT 7%** — ตั้งค่าจด/ไม่จด VAT ต่อ organization, แยกราคา include/exclude VAT, สรุป ภ.พ.30
- **หัก ณ ที่จ่าย (WHT)** — ภ.ง.ด.3/53 สำหรับรายจ่ายบางประเภท
- **e-Tax Invoice & e-Receipt** — เผื่อ integrate ระบบสรรพากรในอนาคต (ออกแบบ document ให้รองรับ)
- รายงาน: รายรับ-รายจ่าย, ภาษีซื้อ-ภาษีขาย, กำไรขาดทุน

## 6. Dashboard (เชื่อมบัญชี)
- ยอดขาย / เงินเข้า / กำไร-ขาดทุน — รวม และแยกราย account/platform/ช่วงเวลา
- ตัวเลขกำไร = ดึงจาก ledger + COGS จริง (ไม่ใช่ประมาณ)

## 7. สิ่งที่ต้องมี test
- weighted-average หลายล็อตต้นทุนต่างกัน
- COGS ของ bundle = ผลรวม component ถูกต้อง
- VAT include/exclude คำนวณตรง, ปัดเศษตามมาตรฐาน
- running number ไม่ซ้ำ/ไม่ข้าม (กันออกเอกสารพร้อมกัน)
- P&L กระทบยอดตรงกับ movement
